# アーキテクチャ概要

このドキュメントは本アプリのアーキテクチャ、依存関係、データフロー、状態管理、テスト方針をまとめたものです。新規参画メンバーが最短で開発に参加できること、将来的な拡張時の設計判断を容易にすることを目的とします。
## 目標
- 一貫した開発体験と可読性の高いコードベース
- 明確な責務分離とテスタビリティの確保
- 機能追加・仕様変更に強い拡張性
- クラッシュやパフォーマンス劣化の早期検知

---

## 全体構成

- プレゼンテーション層: SwiftUI を用いた UI、画面遷移、ユーザー入力の収集
- 状態/ドメイン層: ViewModel（または Presenter/Reducer）による状態管理、ユースケース（アプリケーションサービス）
- データ層: リポジトリ、データソース（ネットワーク、ローカル永続化、キャッシュ）
- 共通基盤: ロギング、エラーハンドリング、DI、Feature Flags、設定

推奨スタイルは MVVM + UseCase（Clean Architecture 準拠）。Swift Concurrency（async/await, actor）を優先的に採用します。
